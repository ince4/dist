# 自我介绍

面试官你好，我叫叶茂，21年杭州电子科技大学毕业之后进入携程工作，在携程火车票部门h5、小程序组做增长营销业务线的开发，主要负责抢票助力和一些营销活动相关的需求。开发中主要使用的技术栈是Vue和node，还有微信小程序。具有一定全栈开发的能力跟经验，

# 未来规划

首先完善丰富一下自己的技术栈，去学习掌握流行的一些技术或者框架等 然后通过实践、阅读源码等方式提升一下深度和广度，提高自身的技术能力。具体方向来说是工程化相关，因为之前一些node开发和优化项目打包速度的经验，希望能把这方面的经验在实践中活用起来，提高一些开发效率

# 为什么离职/空窗期做了啥

业务上过于稳定，没有什么新的点，技术栈也没有什么更新，所以希望到一个新的环境可能更有利于自己的成长。 一方面是休息了一阵子，学画画和日语；另一方面也是趁这个机会学习了一下新的框架和工具，比如vue3、vite、算法数据结构这些，因为之前公司的时候没有机会用

# 兴趣爱好

绘画和外语。 一个是本身比较感兴趣。契机是大学的时候为了学分报了外语课，趁这个机会深入进行了学习，然后也通过了n2考试 绘画就是的兴趣但是因为住校没怎么学，现在买了个数位板学画画，算是弥补一下遗憾吧

# 优缺点

  • 比较认真负责，对于项目遇到的问题都会主动推进、及时沟通交流（项目某个地方遇到阻塞的时候会主动联系相关负责人，并及时同步沟通结果）
  • 负责过一段时间增长业务线的评审跟排期，带领实习生和新同事熟悉开发流程完成开发任务，有一些作为负责人的经验
  • 兴趣广泛，对不同领域有好奇心和学习欲望，能快速找到学习方法和目标，。
  • 具备快速定位和解决问题的能力，保证应用的稳定性。之前携程的有个项目容易遭到微信的封禁，然后通过监控系统和动态生成链接的平台、配置域名等方式，第一时间发现问题并将影响控制在最小
  • 缺点：面试官说部门使用的技术栈以react为主，而我主要的技术栈是vue，react目前会使用但是不够深入；所以针对这点我会深入学习react框架原理，入职的时候根据项目代码能达到不影响开发任务、编写高质量代码的水平。未来希望持续提升知识的广度、深度，并把学到的知识用到实践中去

# 美团、部门

涵盖了外卖、酒店预订、出行、生活服务等多个领域，我主要使用外卖和点评的功能，平常生活中的一部分吧 我了解到这个职位所属的部门是到店事业群的到店餐饮，我理解是为用户提供线上预定、点餐、支付、评价功能，还有涉及到地推、面推的一些相关系统，比如为商家提供上传注册服务，用户到线下门店完成核销这样一个流程。然后我从之前的面试官那边了解到这个职位会涉及到一些系统的重构、跨端开发的一些方面；之前在携程工作中也有一些类似项目开发经历；比如智行车站商家推荐，商家端和用户端的开发，将微信小程序移植到头条，还有h5、微信、app、小程序、服务端不同环境的开发，有一定全栈开发的能力和经验
